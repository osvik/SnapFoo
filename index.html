SnapFoo

SnapFoo is an open source JavaScript library that extends off of SnapSVG to help simplify the SVG animation process.

Getting Started

Download SnapFoo and include it in your project after jQuery and Snap.svg. Once included, call the library for use by passing in the SVG container ID:

Arguments:

  • theSVGContainerID {String}
var snapfoo = snapFoo("#theSVGContainerID");

Animate

The primary animation call that receives an element or group of elements to animate based on theAnimationValues object. Any object that is animated is appended the class "sf#" for a unique identifier.

Arguments:

  • theElement {String}
  • theAnimationValues {Object}
snapfoo.animate(theElement, theAnimationValues)
Return to Top

Animate - theElement {String}

Use a jQuery-like selector to target which elements should be animated. SnapFoo will iterate over any matches found inside of theSVGContainerID.

Target individual elements:

snapfoo.animate("#the-circle", theAnimationValues);

Select numerous elements:

snapfoo.animate(".the-circle, #the-square, .test-element", theAnimationValues);

Select elements by tag:

snapfoo.animate("circle", theAnimationValues);

Use other selector types:

snapfoo.animate("g > circle", theAnimationValues);
Return to Top

Animate - theAnimationValues {Object}

The theAnimationValues objects supports various keys that define the behavior of the animation.

Keys:

  • theAnimationValues {Object}
    • .keyframes {Array}
      • .keyframes.values {Object}
      • .keyframes.duration {Number}
      • .keyframes.easing {Function}
      • .keyframes.options {Object}
        • .options.keyframeCallback {Function}
        • .options.keyframeDelay {Number}
      • .animationDelay {Number}
      • .animationCallback {Function}
      • .loop {Boolean}
      • .loopCount {Number}
      • .stagger {Number}

The minimum requirements include a .keyframes entry with the .values and .duration set. The other keys are set to defaults if not supplied.

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500 }
    ]
};
Return to Top

Animate - keyframes {Array}

The keyframes array stores the information for each step of the animation. Add multiple entries to the keyframes array to create a sequence, with each entry being referred to as a 'frame'.

Values:

  • keyframes {Array}
    • .keyframes.values {Object}
    • .keyframes.duration {Number}
    • .keyframes.easing {Function}
    • .keyframes.options {Object}
      • .options.keyframeCallback {Function}
      • .options.keyframeDelay {Number}
var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500 },
	    { values: { transform: "t100,-40" }, duration: 1500 },
	    { values: { transform: "s2,2", opacity: .75 }, duration: 500, easing: mina.bounce }
    ]
};
Return to Top

Animate - keyframes.values {Object}

The values object supplies the function with the properties that are to be animated. These calls follow the standard syntax of SnapSVG. SnapSVG documentation for transforms can be found here.

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500 },
	    { values: { transform: "t100,-40" }, duration: 1500 },
	    { values: { transform: "s2,2", opacity: .75 }, duration: 500, easing: mina.bounce }
    ]
};
Return to Top

Animate - keyframes.duration {Number}

Set the duration of the keyframe animation in milliseconds.

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500 },
	    { values: { transform: "t100,-40" }, duration: 1500 },
	    { values: { transform: "s2,2", opacity: .75 }, duration: 500, easing: mina.bounce }
    ]
};
Return to Top

Animate - keyframes.easing {Function}

Define the easing type for the keyframe animation. The easing functions are the same as those found in SnapSVG. Examples of supported mina-type easings can be found at this CodePen.

Default: mina.linear

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500, easing: mina.backin },
	    { values: { transform: "t100,-40" }, duration: 1500 },
	    { values: { transform: "s2,2", opacity: .75 }, duration: 500, easing: mina.bounce }
    ]
};
Return to Top

Animate - keyframes.options {Object}

The options object provides additional non-required values to the animation call.

Default: {}

Keys:

  • .keyframes.options {Object}
    • .options.keyframeCallback {Function}
    • .options.keyframeDelay {Number}
var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500, easing: mina.bounce, options: {
		keyframeCallback: function() { console.log("Frame End"); },
		keyframeDelay: 1000
	     }
	 }
    ]
};

Animate - keyframes.options.keyframeCallback {Function}

Supply a function to be executed upon the completion of the specific frame. These callbacks allow for easy chaining and sequencing of multiple animations.

Default: null

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500, easing: mina.bounce, options: {
		keyframeCallback: function() { console.log("Frame End"); }
	        }
	    }
    ]
};
Return to Top

Animate - keyframes.options.keyframeDelay {Number}

The provided keyframeDelay will execute before processing the values of that specific frame.

Default: 0

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500, easing: mina.bounce, options: {
		keyframeDelay: 1000 }
	        }
	    }
    ]
};

Animate - animationCallback {Function}

The animationCallback function executes after completing all keyframe animations.

Default: null

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500, easing: mina.bounce }
     ],
    animationDelay: 2500,
    animationCallback: function() { console.log("All done!"); }
};

Animate - animationDelay {Number}

The animationDelay value is the duration before the first keyframe runs.

Default: 0

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500, easing: mina.bounce }
     ],

Animate - loop {Boolean}

When the loop value is set to true, the series of keyframe animations will repeat.

Default: false

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500, easing: mina.bounce },
	 { values: { transform: "t100,-10 s2,2”, opacity: .75 }, duration: 1500 }
     ],
    loop: true
};
Return to Top

Animate - loopCount {Number}

Setting the loopCount will define how many times the animation loops through the keyframes.

Default: 0 (Infinite)

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500, easing: mina.bounce },
	 { values: { transform: "t100,-10 s2,2”, opacity: .75 }, duration: 1500 }
     ],
    loop: true,
    loopCount: 2
};
Return to Top

Animate - stagger {Number}

The stagger value is a delay placed between elements when animating a series or group.

Default: 0

var theAnimationValues = {
    keyframes: [
            { values: { transform: "t10,40" }, duration: 500, easing: mina.bounce }
     ],
    stagger: 1000

Set the starting frame.

Default: 0

AnimatePath

Animate an element or group of elements along a path. Any object that is animated is appended the class "sf#" for a unique identifier.

Arguments:

  • theElement {String}
  • thePath {String}
  • theDuration {Number}
  • theOptions {Object}
snapfoo.animatePath(theElement, thePath, theDuration, theOptions)
Return to Top

AnimatePath - theElement {String}

Use a jQuery-like selector to target which elements should be animated. SnapFoo will iterate over any matches found inside of theSVGContainerID.

Target individual elements:

snapfoo.animatePath("#the-circle", thePath, theDuration, theOptions);

Select numerous elements:

snapfoo.animatePath(".the-circle, #the-square, .test-element", thePath, theDuration, theOptions);

Select elements by tag:

snapfoo.animatePath("circle", thePath, theDuration, theOptions);

Use other selector types:

snapfoo.animatePath("g > circle", thePath, theDuration, theOptions);
Return to Top

AnimatePath - thePath {String}

Defines the path on which the elements will be animated. Use a unique identifier such as an #ID.

snapfoo.animatePath(“.the-circle”, “#the-path”, 2500, theOptions);
Return to Top

AnimatePath - theDuration {Number}

Set the duration of the path animation in milliseconds.

snapfoo.animatePath(“.the-circle”, “#the-path”, 2500, theOptions);
Return to Top

Define optional path animation behaviors.

Default: {}

Keys:

      • .reverseOptions.delay {Number}

    • theOptions.stagger {Number}

AnimatePath - theOptions.easing {Function}

Define the easing type for the animation. The easing functions are the same as those found in SnapSVG. Examples of supported mina-type easings can be found at this CodePen.

AnimatePath - theOptions.loop {Boolean}

Repeats the path animation. If reverseAtEnd is true, the loop will begin after the element’s reverse animation has completed.

Default: false

    reversePathDirection: true,

Set specific looping behaviors for the animation.

Default: {}

Keys:

    reversePathDirection: true,

AnimatePath - theOptions.loopOptions.loopCount {Number}

Set a limit on the amount of loops that will be completed.

AnimatePath - theOptions.reverseAtEnd {Boolean}

Upon reaching the end of the path, reverseAtEnd will reverse the element(s) along the path back to the original starting point.

Default: false

var theOptions = {
    easing: mina.backin,
    reverseAtEnd: true
};
Return to Top

AnimatePath - theOptions.reverseOptions {Object}

With a reverse animation defined, use the reverseOptions to further customize the behavior.

Default: {}

Keys:

  • reverseOptions {Object}
    • reverseOptions.callback {Function}
    • reverseOptions.delay {Number}
    • reverseOptions.duration {Number}
    • reverseOptions.easing {Function}
var theOptions = {
    reverseAtEnd: true,
    reverseOptions: {}
};
Return to Top

AnimatePath - theOptions.reverseOptions.callback {Function}

Set a different callback for the reverse path animation.

Default: null

var theOptions = {
    reverseAtEnd: true,
    reverseOptions: {

        easing: mina.backout,
        callback: function() { console.log(“Reversed!”); }

Clear

The clear call points to an object containing three functions that will clear various loop intervals.

Functions:

  • .all()
  • .paths()
  • .sequences()
snapfoo.clear.all();
Return to Top

Clear.all()

Clear all stored loop interval IDs. This clear both sequence and path loops.

snapfoo.clear.all();
Return to Top

Clear.paths()

Clear all path loop interval IDs.

snapfoo.clear.paths();
Return to Top

Clear.sequences()

Clear all sequence loop interval IDs created with the .animate() function.

snapfoo.clear.sequences();

Stop

The stop call points to an object containing two functions that will stop an element that is being animated. Note that if the element is part of a loop, the loop will continue until its interval is cleared or its loopCount has been met.

Functions:

  • .sectionAnimations(theSVGContainerID)
  • .specificAnimation(theElement)
snapfoo.stop.sectionAnimations(theSVGContainerID);
Return to Top

Stop.sectionAnimations(theSVGContainerID)

Stops all animations in the current SVG container. Note: The stop() function does not clear any loop intervals.

Arguments:

  • theSVGContainerID {String}
snapfoo.stop.sectionAnimations(theSVGContainerID);
Return to Top

Stop.sectionAnimations(theSVGContainerID) - theSVGContainerID {String}

The ID of the parent SVG container.

Default: theSVGContainerID

snapfoo.stop.sectionAnimations("#section-one");
Return to Top

Stop.sectionAnimations(theElement)

Stop the animation of a specific element. Note: The stop() function does not clear any loop intervals.

Arguments:
